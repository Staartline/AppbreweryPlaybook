We can add javascript to our websites by using DOM. We can incorperate JS into our page using inline JS (onload). This happens inside of our html element. onload(''); will use single quotes. Not modular. Hard to debug, not good in general. Script - > use script tags. It will carry out the JS inside of the tags. External source. script src ="filepath" This means that we can put js into our files and then it will run. We want to put our JS near the bottom of the body page. This is so that we can change elements and define all of them first. After defining all the elements, we can then run our code on it. This is because HTML reads the page from top down. If an element doesn't exist, then values will be null. Best practice is to put the code at the bottom. This lets all the content put the information on screen first, and also helps give an illusion of a faster runtime.
