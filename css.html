What is CSS | Cascading Style Sheets

Cascading Style Sheets is a style language that is used to modify the look of a markup language. This language doesn't work on its own, but modifies the look of an existing file. All browsers have their own default CSS which we can change through our custom CSS.

Before CSS, using HTML elements and attributes can be difficult as lots of code needs to be written to write something simple and making coding errors or spelling mistakes can be easy.

Another way to think about this is that everything on a webpage is made of boxes that we can change, arrange, and style. Pesticide Chrome | Firefox is a useful extension to view the boxes that we are changing.

Comments

Comments are written in CSS using a /**/ format.

EX -> /*comment goes here.*/

Types of CSS

Inline CSS using style attribute

We use inline CSS to write css in a specific HTML's tag using a style attribute. This can be something like changing a background color, or a font.

Ex -> We use a style tag to change the background and the font.

How to set up Inline CSS

  1. Find an element that you would like to inject the CSS into. Ex -> p, h1, h3...
  2. Add in the style attribute. Ex -> p style=""
  3. Add in the desired properties. EX -> p style="font-family:roboto;"
    • Make sure that there is a colon(:) between the property and the value
    • Make sure that the statement ends with a semicolon(;)
    • Ensure that the statement is in double quotes("")
  4. Add in more properties if you want. EX -> p style="background:pink; font-family:roboto;

Our difficulty here is that what if we wanted to change every element on the page? We would have to change all of the elements on the page to contain the same style attribute... There has to be an easier way...

Internal CSS using the style element inside the head element

We use internal CSS when we want to change the way that every element (of the same type) looks on a page. This is done so that we don't need to change every single element individually when we want to update a page's appearance.

How to set up Internal CSS

  1. Find the head element in your HTML document
  2. Insert the style element (< style> < /style>) into the head
  3. Choose an element that you would like to change (EX.body). The format is as follows: body{}
  4. Choose a property that you would like to change. The format is like so: body{background:pink; font-family:roboto;}

External CSS

We use external CSS when you want the style to be applied across the entirety of the website without wanting to copy the internal CSS element to every page on a website.

In short, the external css will modify the appearance of any page that links back to it.

How to set up External CSS

  1. Create a css folder named css. This goes inside of your website's project folder.
  2. Inside of the css folder, create a file called styles.css
  3. Link the styles.css file to your HTML document. Do this by using the link element inside of the head element of your HTML file. (link rel="stylesheets" type="text/css" href="css/styles.css")
    • By the way, if there is a / in /css/styles.css, the / in the front of the directory means that the file is located in the root, instead of the same directory level. If you remove the /, then the file is in the same directory as the current file that is calling it.

Hierarchy

If we specify a CSS rule in two places, which one is applied?

When writing CSS, more specific code overwrites the more general code. If you write a global rule, and then inline CSS, priority is given to inline CSS. With this in mind, you can apply specific changes to the code by writing inline CSS.

Least Specific -> styles.css (External CSS)

More Specific -> style element in the head element (Internal CSS)

Most Specific -> element style="" (Inline CSS)

TIP! It is recommended that you apply all styles in your external CSS (styles.css) file.

Debugging

When code has errors such as logic issues or a spelling mistake, debugging is important!

To debug, you should first try to identify a problem. For example, if you have inserted a CSS file but do not see any CSS, check the link for the css file. If there is an issue where parts of the css page is working but other elements are not, you can try checking for things such as a spelling error or for inline css.

You can use the browser's built in developer tools to try to find issues. Chrome developer tools can be used to find missing files when running an HTML page. Developer tools can also tell us what types of styles are currently running on a page as well.

TIP! To ensure that CSS is working on the current page, make a dramatic change to part of the site's content, such as changing the background-color property, to check.

CSS Syntax

Syntax is the grammer of a programming language, and CSS syntax is no different.

selector {property:value;}

  • selector -> Who are we changing (p, h1, h2...ect.).
  • property -> What are we changing (background-color, font-family, border...ect.). Multiple properties can be applied to the selector.
  • value -> How are we changing the property (black, blue, roboto, dotted...). This depends on the property.

The best practice for css is to put each changed property in alphabetical order. Ex -> p{background-color:orange; color:green; font-family:roboto;}

All CSS Statements must end with a semicolon at the end of the statement.

Selectors can be the following:

  • classes -> .classname{}
    • Used for more specific styling and targets specific elements.
    • Can be used on multiple elements on a page.
    • Can be used when you want to apply the same style to a group of elements.
    • Elements are allowed to have more than one class. Ex.-> You can have a class for an element's color, and for the radius of an image.
  • elements -> h1, p, div
    • Targeting specific element tags will apply a change to every single instance of the tag. Ex -> Applying a red color to an img will apply the red color to every img on the page.
  • ids -> #idname
    • More specific than class selectors.
    • Only one single instance per ID is allowed on a page while you can use classes on multiple places on the page.
    • One ID can only be used for one element (Ie. A single unique element)
    • ID is used sparingly, and it might be suggested against, but there are some cases where it might be useful.

By the way, if you are changing multiple elements, such as h1 and h2, you can format the css like h1,h2{property:value;}

If you mispell the selector or the properties, the CSS might not work correctly.

Each element can only have ONE(1) ID. They can have multiple (MORE THAN ONE) classes, though!

The hierarchy for changes is (least specific) element -> class -> (most specific) ID

Psuedoclasses

Classes can have different states. On occasion, if one would want to hover over the elements on a page, they can change.

For example, if a user were to hover over an image, the background color of the image may change.

EX -> img:hover{property:value}

Properties

How do we know what properties we can affect? There is css reference on MDN.

  • background-> Changes the background color. You can use a color name, or a hexcode to change the color.
  • border-style-> Changes the border's style. This can be a border around an image or a textbox, for instance.
  • height-> This changes the height of our css box.
  • width-> can be done in a numberical or a percentage.
  • Color-> Changes the color of the font
